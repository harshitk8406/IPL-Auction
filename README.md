# 🏏 IPL Auction Simulator

A full-stack web application that simulates the Indian Premier League (IPL) player auction — complete with team management, real-time bidding, player rosters, and budget tracking.

**Live Demo →** *https://ipl-auction-jv77.onrender.com/*

---

## Overview

This project replicates the core mechanics of an IPL-style player auction where multiple team owners compete to build their squad within a fixed budget. It was built as a full-stack project to demonstrate end-to-end development skills — from a REST API with authentication to a responsive React interface, all deployed to the cloud.

---

## Features

- **User Authentication** — Secure register/login flow with JWT-based session management
- **Real-Time Multiplayer Bidding** — Multiple users bid simultaneously via WebSockets; every bid, player change, and budget update is instantly broadcast to all connected clients
- **Auction Synchronization** — All participants see a consistent auction state — no stale views, no missed bids — enforced by server-side Socket.io event management
- **Team Management** — Create and manage IPL franchise teams with individual purse limits
- **Player Auction** — Bid on players in sequence; highest bid wins and deducts from the team's remaining budget
- **Live Budget Tracker** — Real-time display of each team's remaining funds and squad composition
- **Player Database** — Pre-seeded roster of IPL players with roles, base prices, and stats (populated via Python scripts)
- **Persistent Storage** — All auction state is stored in SQLite, surviving server restarts
- **Cloud Deployment** — One-click deployable on Render via `render.yaml`

## Getting Started

### Prerequisites

- Node.js v18+
- Python 3 (optional, for re-seeding player data)

### Installation

```bash
# Clone the repo
git clone https://github.com/Harssue/IPL-Auction.git
cd IPL-Auction

# Install all dependencies (frontend + backend)
npm run setup
```

### Running Locally

```bash
# Build the React frontend
npm run build

# Start the server (serves API + built frontend on one port)
npm start
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Environment Variables

Create a `.env` file inside the `backend/` directory:

```env
JWT_SECRET=your_secret_key_here
DB_PATH=./data.sqlite
NODE_ENV=development
```

## Deployment (Render)

This project includes a `render.yaml` for zero-config deployment on [Render](https://render.com).

1. Fork this repository
2. Create a new **Web Service** on Render and connect your fork
3. Render will automatically detect `render.yaml` and configure the build + start commands
4. The `JWT_SECRET` is auto-generated by Render on first deploy

## How It Works

1. Users register and are assigned to a team (or create one)
2. All participants connect to a shared Socket.io room — the server becomes the single source of truth for auction state
3. An auction admin starts the session — players come up for bidding one at a time, broadcast to every connected client simultaneously
4. When any user places a bid, the server validates it, updates state, and emits a `bid_update` event to all clients — everyone's screen reflects the new highest bid instantly
5. When a player is sold, a `player_sold` event is broadcast: the winning team's purse is deducted in real time across all views
6. Late joiners receive a full state snapshot on connection so they're immediately in sync
7. The auction ends when all players are sold or all teams exhaust their budgets
